## Usage

### Existing Projects
```bash
# Your .gitignore is now comprehensive!
git status  # Should show only tracked files
```

### New Projects
```bash
agenticide chat
You: /stub myapi rust

# Generates:
# - src/myapi/
# - .gitignore (Rust-specific patterns)
# - Cargo.toml
# - etc.
```

### Custom Additions
Add project-specific patterns at the bottom:
```gitignore
# ----------------------------------------------------------------------------
# Custom Patterns (Add project-specific ignores below)
# ----------------------------------------------------------------------------
# Your patterns here
local-config.json
temp-data/
```

## What Gets Ignored

### Always Ignored
- `node_modules/`, `target/`, `build/`, `dist/`
- `*.log`, `*.tmp`, `*.cache`
- `.env*` (environment files)
- `.DS_Store`, `Thumbs.db`
- IDE files (`.vscode/`, `.idea/`)
- Package locks (configurable)
- Secrets (*.key, *.pem)

### Never Ignored (Exceptions)
- `.gitignore` itself
- `.gitkeep` files
- `.github/` directory
- `.devcontainer/` directory
- VSCode workspace settings (optional)

## Testing

```bash
# Check what would be ignored
git status --ignored

# Verify patterns work
touch test.log
git status  # Should not show test.log

# Check untracked files
git clean -n -d -X  # Dry run of ignored files cleanup
```

## Best Practices

1. **Keep custom patterns at bottom** - Easy to find
2. **Comment additions** - Explain why pattern exists
3. **Use wildcards carefully** - Be specific when possible
